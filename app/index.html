CoreOS A lil' Linux for the modern server

Cloud has dramatically changed how we manage compute resources, but we have not changed how we manage Linux itself. CoreOS is a new Linux distro, built specifically for cloud and devops- with security, speed, and automation in mind. We borrow a lot of our philosophies from the browsers. Remember, every browser needs a server. Our the primary design tenets of CoreOS are:

  • Security. The biggest improvement to web security to-date happened when the browsers began shipping automatic updates. Leaning on the frameworks built by ChromeOS, we automatically update the kernel, init system, and base libraries on behalf of the user. Also similar to how the browsers handle sandboxing tabs, we deploy every process into its own container.
  • Speed and Efficiency. Focusing just on server, we are able to get boot times down to under 1000ms. We optimize for memory and disk footprint, currently at 25mb and 50mb respectively. Our kernel is tuned specifically for the most commonly used virtual environments.
  • Automation. Configuration management is a must for any sophisticated architecture, but most users of currently available systems are not satisfied. Our approach is to put REST APIs natively into the OS, in order to more easily build higher level management systems.

Architecture

CoreOS is a fork of ChromiumOS. We started by replacing ChromiumOS's upstart based init system with systemd, and then tuned the packages required for server. The automatic update mechanism is a minimally modified version of what ships with ChromiumOS. From there, we layer in our own golang based REST APIs for managing the host, which interact with systemd via D-Bus.
